# تكسي واحد عراق | Taxi One Iraq

منصة طلب سيارات أجرة للعراق — تطبيقان + خادم:

| المكوّن | المسار | الحالة |
|---|---|---|
| 📱 تطبيق الزبون (Android) | [`android/app/`](android/app/) | يعمل — APK جاهز من CI |
| 🚗 تطبيق السائق (Android) | [`android/driver/`](android/driver/) | يعمل متصلاً بالخادم — APK من CI |
| ⚙️ الخادم Backend | [`backend/`](backend/) | FastAPI: حسابات OTP، رحلات، تتبع، دفع |
| 🍎 تطبيق الزبون (iPhone) | [`IraqTaxi/`](IraqTaxi/) | نموذج SwiftUI أولي |

## هوية المنتج

- الاسم الظاهر: **تكسي واحد عراق**
- اللغة الأساسية: العربية (RTL)، مع English.
- نطاق الخدمة: جميع محافظات العراق.
- الخرائط: **OpenStreetMap مجاناً داخل التطبيق + الملاحة عبر Waze للسائق** (زر واحد يفتح ويز على وجهة الرحلة).
- اختيار نقطة الانطلاق: خريطة بملء الشاشة مع دبوس وعنوان عربي تلقائي (Nominatim) والأسعار معروضة مقدماً لكل فئة.
- حوافز السائقين: سلّم مكافآت يومي بمؤشر تقدم وعدّاد وقت، قابل للتخصيص من الخادم (`INCENTIVE_PLANS`).
- **لوحة تحكم المالك** على `/admin`: نسبة العمولة، رسوم الحجز، وتعرفة كل فئة — تُعدَّل من المتصفح وتسري فوراً دون تحديث التطبيقات.

## لوحة التحكم (الرسوم والتسعير)

1. ولّد رمز إدارة وضعه في `backend/.env`:
   `python -c "import secrets;print(secrets.token_urlsafe(24))"` ← `ADMIN_TOKEN=...`
2. افتح `https://خادمك/admin` من أي متصفح (يعمل من الهاتف).
3. أدخل الرمز ← عدّل **نسبة العمولة %** و**رسوم الحجز** و**تعرفة الفئات** ← حفظ.
4. تُطبَّق على الرحلات الجديدة فوراً، وتُخصم العمولة تلقائياً من أرباح السائق (يرى الصافي والعمولة في تطبيقه).
- الدفع: نقدي + زين كاش + FIB (يتفعّل بوضع مفاتيح التاجر) — باي بال غير متاح للتجار في العراق.

## تحميل ملفات APK

كل دفعة تبني التطبيقين تلقائياً عبر GitHub Actions:

1. افتح تبويب **Actions** ← آخر تشغيل ناجح لـ **Android APK**.
2. حمّل من قسم Artifacts:
   - `IraqTaxi-rider-release-apk` — تطبيق الزبون
   - `IraqTaxi-driver-release-apk` — تطبيق السائق
3. انقل الملف للهاتف وفعّل «التثبيت من مصادر غير معروفة».

## تجربة المنظومة كاملة (زبون + سائق + خادم)

```bash
# 1) شغّل الخادم
cd backend
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
cp .env.example .env          # OTP_ECHO=true يعرض رمز الدخول داخل التطبيق للتجربة
.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000

# 2) ثبّت تطبيق السائق (نسخة debug تسمح بالاتصال بخادم محلي HTTP)
#    في شاشة الدخول ضع عنوان الخادم: http://<IP-جهازك>:8000
#    (أو http://10.0.2.2:8000 من داخل المحاكي)
```

سجّل دخول سائق، فعّل «متاح»، ثم أنشئ رحلة عبر `http://localhost:8000/docs` أو من تطبيق زبون متصل — سيظهر الطلب للسائق خلال ثوانٍ ويستطيع قبوله وقيادته حتى الإكمال بملاحة Waze.

## حماية المستخدم وبياناته

- **تطبيق الزبون يستخدم الإنترنت للخرائط والعناوين فقط** — بلا إعلانات، بلا متتبعات، وأذونات التخزين تُزال قسرياً حتى لو طلبتها مكتبة خارجية.
- **الموقع أثناء الاستخدام فقط** في التطبيقين — يتوقف تلقائياً عند مغادرة التطبيق، ولا يُطلب إذن الخلفية أبداً.
- **رموز الدخول وسجل الرحلات مشفّرة AES-256-GCM** بمفتاح داخل Android Keystore.
- **OTP بلا كلمات مرور**: رموز مؤقتة (5 دقائق) مخزّنة كـ HMAC، بحد 5 محاولات و5 إرسالات بالساعة.
- **JWT** موقّعة للجلسات، مع فصل صارم بين صلاحيات الزبون والسائق.
- **مفاتيح الدفع على الخادم فقط** — لا تدخل الـ APK أبداً.
- **HTTPS إجباري في نسخ release** (السماح بـ HTTP محصور بنسخة debug للتطوير المحلي).
- منع النسخ الاحتياطي السحابي + زر «حذف جميع بياناتي» + R8 minify.

## التوقيع للنشر

نسخ release موقّعة حالياً بمفتاح debug للتجربة. قبل النشر:

```bash
keytool -genkeypair -v -keystore release.jks -keyalg RSA -keysize 4096 -validity 10000 -alias taxione
```

ثم أضف `signingConfig` يقرأ من `keystore.properties` (خارج Git). لا ترفع المفتاح للمستودع أبداً.

## ما تبقى قبل الإطلاق التجاري

1. **عقود الدفع**: تاجر زين كاش + FIB (+ Qi لسوبر كي) — الكود جاهز، ضع المفاتيح في `backend/.env`.
2. **مزود SMS عراقي** لرموز OTP (واجهة جاهزة في `backend/app/sms.py`).
3. نشر الخادم على HTTPS + PostgreSQL، مع `OTP_ECHO=false` و`DRIVER_AUTO_APPROVE=false`.
4. ربط تطبيق الزبون بالخادم (تطبيق السائق مربوط بالفعل)، وخريطة اختيار الوجهة.
5. لوحة إدارة لاعتماد السائقين بعد فحص المستمسكات، والدعم، والتسعير.
6. سياسة خصوصية وشروط استخدام + زر طوارئ داخل الرحلة.
